4 Что означают показатели в выводе команды top?
    Начиная с верхней строчки:
  • отображение текущего времени
  • время прошедшее от старта
  • количество активных сессий (1 user)
  • KiB Mem|Swap секции использования памяти:
    всего, свободно, занято, кэш обращения к диску, общий объем
    • Tasks:(Задачи)
    • total - общее кол. процессов в ОС>
    • running -процессы выполняеме CPU
    • sleep - процессы ожидающие завершения (Interruptible sleep),
      процесы ожидающие завершения ввода/вывода
    • stopped - остановленные процессы
    • zombie - структуры данных с завершенными дочерними процессами
    • %Cpu(s) распредение в % загрузки процессора:
    • us - в пользовательском пространстве
    • sy - в пространстве ядра
    • ni - процессы с ручной установкой "nice" параметра,
      ранжирование приоритетов исполнения
    • id бездействие (idle)
    • wa ожидание завершения ввода/вывода (waiting for I/O)
      • затраты на:
      • hi - hardware interrups, прерывания оборудования
      • si - software interrups, прерывания ПО
      • st - "steale", на обслуживание виртуалзации
  • load average - усредненная загрузка за: 1 мин., 5 мин., 15 мин.
    • Обобщенная таблица
    • PID - process ID
    • USER - `эфективное` имя пользователя запустившего процесс
    • PR - отображение приоритета процесса относительно ядра
    • NI - отображение значения `nice` у процесса
    • VIRT - общее кол. памяти занятое процессом
    • RES - размер задействованный в RAM памяти
    • %MEM - процент доступной памяти
    • SHR - память, используемая несколькими процессами
      • S - сотояние процесса одной буквой:
      • Runnable (R)
      • Interruptible sleep (S)
      • Uninterruptible sleep (D)
      • Stopped (T)
      • Zombie (Z))
    • TIME+ - общее время затраченное на процесс со времени старта
    • COMMAND - название процесса
